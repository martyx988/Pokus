## T29: Universe-Change Event Recording for M2 Lifecycle Events

### Purpose
Record auditable universe-change history for additions, exclusions/degradations/restorations, removals, and symbol/name/identifier changes detected in M2 flows.

### Scope
Implement lightweight universe-change event writer and trigger points from discovery/ranking/support-state updates.

### Allowed Changes
- Universe-change event model/service modules
- Event-type constants for required M2 event categories
- Focused tests for reason/detail payload completeness

### Forbidden Changes
- Do not implement operator dashboard UI or broad reporting features.
- Do not implement publication denominator logic.
- Do not add new provider behaviors.

### Inputs
- `design_phase/product_spec.md` FR-9, FR-9a, AC-22, AC-23
- State transitions from T25 and T28

### Relevant Context
Milestone 2 requires auditable universe-change history and explainable reasons for support-state movement and identity changes.

### Implementation Budget

- Behavior count: 1 behavior (emit lifecycle history events with reasons/details).
- Model/API/workflow count: 1 universe-change model/path, 0 app APIs, 1 event-recording workflow.
