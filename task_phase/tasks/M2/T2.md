## T2: Launch Exchange and Launch Instrument-Type Baseline Records

### Purpose
Ensure launch defaults for NYSE, Nasdaq, Prague Stock Exchange, and launch instrument types (stocks/ETFs/ETNs) are present and selectable.

### Scope
Create deterministic baseline records and mapping constants for launch exchanges/types without running discovery.

### Allowed Changes
- Exchange/type reference data modules or migrations under `project/`
- Safe bootstrap/seed command used by admin/worker setup
- Focused tests for baseline idempotent creation

### Forbidden Changes
- Do not implement discovery jobs or listing ranking.
- Do not implement calendar validation logic beyond baseline linkage fields.
- Do not expose app-facing endpoints.

### Inputs
- `design_phase/product_spec.md` (AC-1, FR-56a)
- `task_phase/roadmap_plan.md` Milestone 2 scope
- Output of T1

### Relevant Context
Launch requires NYSE, Nasdaq, and PSE in scope and the same trust bar. Discovery and scheduling need canonical exchange/type records first.

### Implementation Budget
- Production files: max 1-3 reference-data files.
- Behavior count: 1 behavior (idempotent baseline population).
- Model/API/workflow count: up to 2 reference model touches, 0 app APIs, 0 discovery workflows.
- Test scope: idempotent creation and exact expected launch records.

### Expected Output
Baseline exchange/type records exist and match launch scope naming/codes used across M2 tasks.

### Acceptance Criteria
- NYSE, Nasdaq, and PSE records are created or confirmed.
- Stocks, ETFs, ETNs instrument-type records are created or confirmed.
- Baseline population can run multiple times without duplicates.

### Test Requirements

- Assertion test for exact launch baseline records.
