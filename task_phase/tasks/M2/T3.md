# T3: Launch Exchange Calendar and Market-Closed Decisions

## Purpose
Implement launch exchange calendar behavior needed to identify expected trading days and market-closed days.

## Scope
- Wire launch exchanges to the existing calendar abstraction.
- Support weekend and holiday market-closed decisions for NYSE, Nasdaq, and PSE.
- Support manual override records for special closures or library gaps if the Milestone 1 calendar foundation includes override storage.
- Return a clear expected-trading-day decision for a given exchange and exchange-local date.

## Allowed Changes
- Calendar service/module and launch exchange calendar mapping.
- Calendar override/reference-data models only if already planned by the Milestone 1 foundation.
- Calendar tests and fixtures.

## Forbidden Changes
- Do not implement provider/calendar validation reports for exchange admission; that belongs to Milestone 3.
- Do not add custom calendar adapters unless the existing calendar abstraction requires a minimal adapter wrapper.
- Do not trigger price computation or publication from calendar checks.
- Do not change public readiness semantics.

## Inputs
- T1 launch exchange configuration.
- Milestone 1 calendar abstraction.

## Relevant Context
Market-closed handling must prevent price computation for closed exchange days. Architecture recommends using an exchange-agnostic calendar library first, with manual overrides for holidays, special closures, late opens, or source gaps. Calendar mistakes can cause false failures later.

## Expected Output
The system can answer whether each launch exchange is expected to trade on a given local date and can identify market-closed days.

## Acceptance Criteria
- Calendar lookup works for `NYSE`, `NASDAQ`, and `PSE`.
- Weekends are treated as closed for launch exchanges.
- Known holidays in test fixtures are treated as closed.
- Known trading days in test fixtures are treated as expected trading days.
- Manual override behavior is covered if override storage exists.

## Test Requirements
- Unit tests for weekends, holidays, and regular trading days.
- Tests proving PSE calendar lookup is supported or clearly raises a controlled configuration error that blocks later validation.
- Override precedence tests if overrides exist.

## Dependencies
- T1.

## Stop Condition
Stop when launch exchange calendar decisions are deterministic and test-covered.
