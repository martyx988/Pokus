# T9: Primary and Best-Listing Selection Policy

## Purpose
Select one primary/best listing from candidate listings using the required global ranking order.

## Scope
- Implement ranking by home exchange, then highest turnover, then exchange activity priority.
- Preserve ranking inputs and selection reason for explainability.
- Mark non-selected duplicate candidates as not app-visible through support-state updates or selection output.
- Handle missing ranking inputs through explicit exclusion or needs-review reasons.

## Allowed Changes
- Universe selection/ranking service.
- Supported universe state updates needed to record chosen and non-chosen listings.
- Selection reason fields or related metadata.
- Focused unit tests for ranking decisions.

## Forbidden Changes
- Do not implement provider validation or price ingestion.
- Do not create public API endpoints.
- Do not manually override the ranking order.
- Do not merge unrelated lifecycle/exclusion rules into the ranking service beyond selection outcomes.

## Inputs
- T5 candidate listings and identifiers.
- T6 supported universe state.
- T8 exchange activity priority.

## Relevant Context
A company listed on multiple exchanges should generally be represented by one supported listing. The required ranking order is home exchange, highest turnover, then automatically derived exchange activity priority. App-facing APIs must not expose duplicate unsupported candidates.

## Expected Output
The system can choose a single supported listing from related candidates and explain why that listing won.

## Acceptance Criteria
- Home-exchange candidates win over otherwise comparable non-home candidates.
- Higher turnover wins when home-exchange precedence does not decide.
- Exchange activity priority is used only after home exchange and turnover.
- Selection reason and ranking inputs are stored or returned for audit.
- Non-selected duplicate listings are not app-visible.

## Test Requirements
- Unit tests for each ranking tier.
- Tie and missing-data tests.
- Tests that selection output updates or produces supported-state changes without exposing non-selected candidates.

## Dependencies
- T5.
- T6.
- T8.

## Stop Condition
Stop when best-listing selection is deterministic, explainable, and covered by ranking tests.
