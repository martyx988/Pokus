## T63: Persist Universe Source Validation Results and Classification Records

### Purpose
Create durable storage for real source-validation evidence, per-source verdicts, and source-role outcomes used by Milestone 3.1.

### Scope
Add the smallest schema and persistence layer needed to record live universe-source validation runs, measured evidence, source classifications, and role assignments.

### Allowed Changes
- `project/src/pokus_backend/domain/**` source-validation models or tightly related persistence helpers
- `project/src/pokus_backend/validation/**` result-writing helpers
- `project/migrations/**` for any required schema change
- `project/tests/**` focused model/persistence tests
- `task_phase/tasks/M3.1/T63.md` status updates on completion

### Forbidden Changes
- Do not add live provider calls yet.
- Do not implement the combined loader yet.
- Do not add price-loading, publication, or app-read behavior.

### Relevant Context
Milestone 3.1 requires real source testing with repo-tracked evidence. The system needs explicit storage for source verdicts like `promote`, `fallback only`, `validation only`, `not for universe loader`, and `reject`, plus enough measured context to explain those decisions later.

### Implementation Budget
- Production files: 2-4 closely related model/persistence files plus 1 migration if needed.
- Behavior count: 1 behavior (persist per-source validation evidence and classification state).
- Model/API/workflow count: 1 tightly related model family, 0 APIs, 1 persistence boundary.
- Test scope: schema and persistence invariants for source evidence/classification records.

### Expected Output
A durable source-validation record exists for live Milestone 3.1 runs, with explicit verdicts, measured fields, and role-assignment support.

### Acceptance Criteria
- The system can persist a source-validation record keyed to source identity and validation run context.
- Persisted evidence can represent availability, auth requirement, rate-limit/quota notes, speed notes, exchange-coverage notes, and role/verdict fields.
- Classification values required by Milestone 3.1 are represented explicitly and validated.

### Test Requirements
- Focused persistence/model tests covering valid source verdicts, required evidence fields, and invalid classification rejection.

### Dependencies

### Stop Condition
Stop when live source evidence and classification records can be stored and read back consistently.
