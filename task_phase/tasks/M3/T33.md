## T33: Provider Attempt Logging for Price Fetch Calls

### Purpose
Persist provider attempt evidence for price-fetch calls, including latency, error/rate-limit outcomes, and missing/stale indicators.

### Scope
Implement provider-attempt write path and minimal read helper for investigation support. No candidate-value selection behavior.

### Allowed Changes
- Provider attempt persistence modules under `project/`
- Focused model/migration updates for provider-attempt fields only
- Worker-side logging service invoked from adapter-call boundary
- Focused persistence tests

### Forbidden Changes
- Do not implement selected-value decision logic.
- Do not implement validation reporting workflows.
- Do not expose app endpoints.

### Inputs
- T32 normalized provider candidate contract
- `design_phase/product_spec.md` FR-19b, FR-19c
- `design_phase/architecture.md` ingestion/observability expectations

### Relevant Context
Milestone 3 requires durable provider-call evidence so failures and source-quality issues are explainable before publication workflows rely on providers.

### Implementation Budget
- Production files: max 3-5 provider-attempt focused files.
- Behavior count: 1 behavior (persist provider attempt evidence).
- Model/API/workflow count: 1 model family, 0 APIs, 1 worker logging behavior.
- Test scope: attempt persistence completeness and idempotent correlation behavior.

### Expected Output
Provider call attempts are stored with latency, status, error/rate-limit markers, and contextual metadata needed for later validation analysis.

### Acceptance Criteria
- Attempt records capture provider, exchange, instrument/listing context, request type, timing, and outcome classification.
- Error, rate-limit, stale, and missing outcomes are representable and persisted.
- Logging path can be invoked repeatedly without duplicate-collision instability for the same attempt key strategy.

### Test Requirements
- Persistence tests for success and failure attempt records.
- Tests for rate-limit and missing/stale outcome flags.
