## T6: Provider Registry and Attempt Schema

### Purpose
Create the schema for provider records and provider attempt evidence.

### Scope
Add `Provider` and `ProviderAttempt` models/migrations for later market-data adapter evidence, including latency, errors, rate limits, stale/missing-value markers, and normalized metadata storage.

### Allowed Changes
- Domain model files for providers and attempts
- Migrations for provider evidence tables
- Focused model tests

### Forbidden Changes
- Do not implement real provider adapters or source-prioritization logic.
- Do not store selected price values in this task.
- Do not perform external network calls.

### Inputs
- T3 exchange schema
- Product requirements for multiple free/free-tier providers and retained attempt evidence

### Relevant Context
Provider viability is a later milestone risk, but Milestone 1 must reserve provider records. Attempts must preserve enough evidence for later validation, source prioritization, debugging, and operator investigation.

### Implementation Budget
- Production files: expected maximum 1-3 closely related domain/migration files.
- Behavior count: 1 behavior, durable provider attempt evidence.
- Model/API/workflow count: 2 model entities, 0 APIs, 0 workflows.
- Test scope: provider uniqueness, attempt status/error metadata, exchange association, and timing fields.

### Expected Output
Provider and provider-attempt tables exist with enough evidence fields for later ingestion and validation work.

### Acceptance Criteria
- Providers have stable identifiers and active/configuration status.
- Attempts can record provider, exchange, request purpose/load type, timing, latency, result status, error details, rate-limit indication, and normalized metadata.
- Attempt records do not decide final price values.

### Test Requirements
Test provider uniqueness and required attempt fields for success, timeout/error, and rate-limit evidence.

### Dependencies
T3.

### Stop Condition
Stop when provider evidence schema is migrated and tested without adapters.
