## T10: Publication and Quality Check Schema

### Purpose
Create the schema for exchange/day publication state and quality validation evidence.

### Scope
Add `PublicationRecord` and `QualityCheck` models/migrations for exchange/day publication status, coverage summary, correctness validation result, benchmark mismatch evidence, and publication timestamps.

### Allowed Changes
- Domain model files for publication and quality checks
- Migrations for publication/quality tables
- Focused model tests

### Forbidden Changes
- Do not implement publication readiness algorithms.
- Do not implement read-model refresh.
- Do not expose app readiness endpoints.

### Inputs
- T9 exchange-day load schema
- Product publication rules for independent exchange/day readiness

### Relevant Context
Each exchange publishes independently. Current-day opening-price publication is not ready until eligible instruments have terminal outcomes, coverage exceeds 99%, and correctness validation succeeds. Milestone 1 reserves the state and evidence shape only.

### Implementation Budget
- Production files: expected maximum 1-3 closely related domain/migration files.
- Behavior count: 1 behavior, durable publication and quality evidence representation.
- Model/API/workflow count: 2 model entities, 0 APIs, 0 workflows.
- Test scope: status constraints, relationship to exchange-day load, coverage fields, and correctness result fields.

### Expected Output
Publication and quality evidence tables exist and can later support readiness decisions.

### Acceptance Criteria
- Publication records reference an exchange/day load.
- Publication status can represent unpublished, ready, failed/blocked, and market-closed outcomes needed for later flows.
- Coverage, eligible count, success count, failed count, and correctness-validation result fields are representable.
- Quality checks can store benchmark mismatch percentage and summary evidence without running validation.

### Test Requirements
Test status constraints, uniqueness per exchange/day/load type where applicable, and required quality/publication fields.

### Dependencies
T9.

### Stop Condition
Stop when publication and quality schemas are migrated and tested without readiness behavior.
