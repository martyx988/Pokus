## T005: Provider Attempt and Price Storage Models

### Purpose
Create durable storage for provider records, provider attempts, and selected daily price records.

### Scope
Implement model fields, constraints, migrations, admin registration, and tests for:
- `Provider`
- `ProviderAttempt`
- `PriceRecord`

### Allowed Changes
- `project/apps/domain/models/`
- `project/apps/domain/admin.py`
- `project/apps/domain/migrations/`
- `project/apps/domain/tests/`

### Forbidden Changes
- Do not implement provider adapters or make external network calls.
- Do not implement source-prioritization logic.
- Do not compute adjusted closes, opening prices, signals, or quality checks.
- Do not add API endpoints.

### Inputs
- T003 core identity models

### Relevant Context
Historical records use adjusted daily closes. Current-day records use official exchange-local unadjusted opening prices. Every persisted price record stores currency. Provider attempts preserve audit evidence for candidate values and failures before a selected source wins.

### Expected Output
The schema can store generic providers, per-provider call/result evidence, and one selected price per listing/trading day/price type.

### Acceptance Criteria
- `PriceRecord` supports at least historical adjusted close and current-day open price types.
- `PriceRecord` includes value, currency, exchange-local trading date, provider reference, and observation timestamp where available.
- A uniqueness constraint prevents duplicate selected prices for the same listing, date, and price type.
- `ProviderAttempt` can record success, failure, timeout/rate-limit evidence, candidate value, and raw metadata references without storing secrets.
- Tests cover price uniqueness and currency requirement.

### Test Requirements
- Model tests for price-type choices, required currency, and uniqueness.
- Model tests for provider attempt statuses and linkage to provider/listing/date/load type.

### Dependencies
T003.

### Stop Condition
Stop when price and provider-attempt storage exists without provider integration behavior.
