## T015: Structured Logging and Metrics Plumbing

### Purpose
Add initial structured logs and metrics helpers for job lifecycle and admin-command events.

### Scope
Configure structured logging, define event helper functions, and add lightweight metrics counters/gauges/histogram placeholders.

### Allowed Changes
- `project/config/settings/`
- `project/apps/observability/`
- `project/apps/jobs/` only to call logging helpers around state changes
- `project/apps/operations/` only to call logging helpers around admin-command records
- `project/apps/observability/tests/`
- `project/apps/jobs/tests/`
- `project/apps/operations/tests/`

### Forbidden Changes
- Do not add paid or external observability vendors.
- Do not log credentials, tokens, provider keys, or raw secrets.
- Do not implement provider, signal, publication, or ingestion metrics beyond placeholders.
- Do not refactor existing domain models except to emit logging events for already implemented state changes.

### Inputs
- T006 load job model
- T011 admin command foundation

### Relevant Context
The architecture requires structured logs for job start/finish/failure and admin commands in Milestone 1. Later logs will cover provider attempts, source conflicts, publication decisions, quality checks, signals, reprocessing, and universe changes.

### Expected Output
The codebase has a consistent structured logging/metrics foundation that job and admin-command code can use immediately.

### Acceptance Criteria
- Job lifecycle events produce structured logs with job id, idempotency key, state, timestamps, and error reason when present.
- Admin-command events produce structured logs with command id, command type, status, actor reference, and reason.
- Metrics helpers expose placeholders for API response latency/error rate, job retry count, queue depth, worker heartbeat, and exchange/day status.
- Tests verify expected structured fields and secret redaction behavior.

### Test Requirements
- Unit tests capturing log records for job and admin-command events.
- Unit tests for redacting or avoiding secret-like fields.
- Unit tests for metrics helper registration or no-op behavior in test mode.

### Dependencies
T006, T011.

### Stop Condition
Stop when logging and metrics helpers exist and are tested without external observability services.
