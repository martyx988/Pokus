## T016: Milestone 1 Integration Gate

### Purpose
Validate that the Milestone 1 foundation works as a coherent deployable slice.

### Scope
Add focused integration tests and documentation that prove the API role, worker role, PostgreSQL schema, job semantics, health checks, and observability plumbing fit together.

### Allowed Changes
- `project/tests/`
- `project/apps/*/tests/`
- `project/README.md`
- `project/docker-compose.yml` only for test-related fixes
- `project/config/settings/` only for test-related fixes

### Forbidden Changes
- Do not add new production behavior beyond fixing defects found while validating M1.
- Do not implement provider ingestion, real publication readiness, real read-model builds, or signal generation.
- Do not alter design documents or task files.
- Do not expand the milestone scope.

### Inputs
- T001 through T015

### Relevant Context
The validation goal is to prove API and worker processes can share durable PostgreSQL state, create and transition jobs idempotently, expose health, and represent exchange/day lifecycle states without performing real provider ingestion.

### Expected Output
Milestone 1 has a repeatable validation command and passing tests for the foundation.

### Acceptance Criteria
- API and worker settings both connect to the same PostgreSQL configuration.
- Migrations apply cleanly in a fresh database.
- Job state transition and idempotency tests pass.
- Health checks report API, database, worker/scheduler heartbeat, queue age/depth, and backup placeholder state.
- Structured logs include job lifecycle and admin-command events.
- README documents the commands used for local validation.

### Test Requirements
- Full test suite for Milestone 1.
- Integration test for database-backed API/worker shared state where practical.
- Migration application test or documented migration check command.

### Dependencies
T001, T002, T003, T004, T005, T006, T007, T008, T009, T010, T011, T012, T013, T014, T015.

### Stop Condition
Stop when all M1 validation commands pass or any remaining failure is documented as an explicit blocker.
