## T5: Universe Change Audit Schema

### Purpose
Create the audit schema for supported-universe lifecycle events.

### Scope
Add `UniverseChangeRecord` storage for additions, removals, exclusions, delisting suspicion, restoration, degradation, symbol/name/identifier changes, and related old/new state evidence.

### Allowed Changes
- Domain model files for universe-change audit records
- Migrations for the audit table
- Focused model tests

### Forbidden Changes
- Do not implement automatic universe maintenance workflows.
- Do not implement operator dashboard views or filters.
- Do not change support-state selection behavior from T4.

### Inputs
- T4 instrument identity and support-state schema
- Product requirements for lightweight universe-change history

### Relevant Context
The operator must later investigate additions, removals, exclusions, delisting suspicion, restorations, degradation, and symbol/name/identifier changes. Milestone 1 only reserves the durable audit shape.

### Implementation Budget
- Production files: expected maximum 1-2 closely related domain/migration files.
- Behavior count: 1 behavior, durable recording shape for universe-change evidence.
- Model/API/workflow count: 1 model entity, 0 APIs, 0 workflows.
- Test scope: required fields, event-type constraints, and references to instrument/listing/exchange context.

### Expected Output
Universe-change records can be stored with event type, effective day, reason, optional details, and old/new state evidence.

### Acceptance Criteria
- Required event types from the specification are representable.
- Records can reference the relevant instrument, listing, exchange, and instrument type context where available.
- Reason is required; details may be optional.
- Old and new state evidence can be stored for later operator reporting.

### Test Requirements
Test valid event types, required reason handling, and reference integrity.

### Dependencies
T4.

### Stop Condition
Stop when the universe-change audit schema exists and passes focused model tests.

### Status
Done.

### Completion Summary
Implemented `UniverseChangeRecord` audit schema in the domain model and migration with required lifecycle event types (`added`, `removed`, `excluded`, `delisting_suspected`, `restored`, `degraded`, `symbol_changed`, `name_changed`, `identifier_changed`), required `reason`, optional `details`, and `old_state_evidence`/`new_state_evidence` JSON fields. Added optional context references to instrument, listing, exchange, and instrument type. Added focused model tests validating event-type representability, required/non-empty reason handling, reference context linking, and foreign-key reference integrity.
