## T12: Admin Command and Audit Record Schema

### Purpose
Create durable records for restricted administrative actions and audit evidence.

### Scope
Add `AdminCommand` and general `AuditRecord` schema for later configuration changes, validation triggers, historical reprocessing requests, retention actions, and manual job actions with required reasons.

### Allowed Changes
- Domain model files for admin commands and audit records
- Migrations for command/audit tables
- Focused model tests

### Forbidden Changes
- Do not implement admin APIs or operator UI.
- Do not execute reprocessing, retention, retry, cancel, or mark-failed actions.
- Do not implement authentication behavior in this task.

### Inputs
- T8 job schema where manual job actions will later apply
- Product requirements for audited admin/operator actions

### Relevant Context
Administrative capabilities must be restricted and audited. Manual job retry, cancel, and mark-failed actions require recorded reasons. Historical reprocessing and retention are explicit actions and must preserve audit evidence.

### Implementation Budget
- Production files: expected maximum 1-3 closely related domain/migration files.
- Behavior count: 1 behavior, durable command/audit representation.
- Model/API/workflow count: 2 model entities, 0 APIs, 0 workflows.
- Test scope: command type constraints, required reason fields, actor metadata, and audit linkage.

### Expected Output
Admin commands and audit records can be stored for later restricted operator/admin workflows.

### Acceptance Criteria
- Command types can represent configuration, validation, reprocessing, retention, retry, cancel, and mark-failed intents.
- Commands that mutate operations can store a required reason.
- Audit records include actor/request metadata fields without storing secrets.
- Records can reference related jobs or domain objects generically or through explicit supported links.

### Test Requirements
Test command type validation, reason requirements for mutating commands, and safe audit metadata storage.

### Dependencies
T8.

### Stop Condition
Stop when command/audit schema exists and tests pass without executing commands.
