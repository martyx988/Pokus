## T2: Add PostgreSQL and Local Runtime Configuration

### Purpose
Make the API and worker roles run against the same PostgreSQL database in a local Docker Compose/VPS-aligned setup.

### Scope
- Add local PostgreSQL configuration.
- Add environment variable handling for database, secret key, debug mode, allowed hosts, and role-specific runtime settings.
- Add Docker Compose services for PostgreSQL, API, and worker/scheduler.
- Add a launch-safe `.env.example` without real secrets.

### Allowed Changes
- `project/docker-compose.yml`
- `project/Dockerfile`
- `project/.env.example`
- `project/config/settings/**`
- `project/README.md`
- Tests for database settings resolution.

### Forbidden Changes
- Do not store real secrets, credentials, provider keys, or production passwords.
- Do not introduce managed cloud database configuration as the default.
- Do not introduce Redis or non-PostgreSQL brokers.
- Do not implement backups beyond placeholders/config hooks required by later health checks.
- Do not change task files from other milestones.

### Inputs
- T1 project skeleton.
- `design_phase/architecture.md` sections on PostgreSQL, deployment/hosting, sensitive data handling, and background jobs.
- `task_phase/roadmap_plan.md` Milestone 1 acceptance criteria.

### Relevant Context
PostgreSQL is the system of record for domain data, job state, publication state, and audit records. Initial deployment is a single VPS using Docker Compose. The API and worker must share durable state.

### Expected Output
- Local Compose can start PostgreSQL and define separate API and worker services.
- Django database settings point to PostgreSQL through environment variables.
- The repository contains safe example configuration for local development.

### Acceptance Criteria
- `docker compose config` succeeds from `project/`.
- API and worker services reference the same database service and database name.
- `.env.example` contains placeholders only.
- Django can resolve database settings from environment variables.

### Test Requirements
- Add a settings test that verifies PostgreSQL engine configuration when env vars are supplied.
- Add a documentation check or command note for running migrations against the Compose database.

### Dependencies
T1.

### Stop Condition
Stop when a developer can start the shared database and both runtime roles have a clear, common database configuration.
