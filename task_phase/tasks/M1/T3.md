## T3: Exchange and Instrument Type Schema

### Purpose
Create the reference schema for market publication units and supported instrument categories.

### Scope
Add models and migrations for `Exchange` and `InstrumentType`, including the future synthetic `CRY` exchange capability and launch instrument-type categories.

### Allowed Changes
- Domain model files for reference data
- Migrations for exchange and instrument type tables
- Focused model tests

### Forbidden Changes
- Do not implement admin configuration workflows.
- Do not implement universe discovery, listing selection, calendar behavior, or crypto admission logic.
- Do not add price, signal, publication, or job tables.

### Inputs
- T2 database baseline
- Product launch scope: NYSE, Nasdaq, Prague Stock Exchange; stocks, ETFs, ETNs; future crypto under `CRY`

### Relevant Context
An exchange is the exchange/day publication unit. Instrument types include launch stocks, ETFs, and ETNs, while crypto is excluded from launch but reserved for future `CRY` support.

### Implementation Budget
- Production files: expected maximum 1-3 closely related domain/migration files.
- Behavior count: 1 behavior, durable reference representation for exchanges and instrument types.
- Model/API/workflow count: 2 model entities, 0 APIs, 0 workflows.
- Test scope: field constraints, uniqueness, and valid reference categories.

### Expected Output
`Exchange` and `InstrumentType` schema exists with constraints suitable for later admin scope and publication work.

### Acceptance Criteria
- Exchanges have stable codes/names and can represent launch exchanges and future synthetic `CRY`.
- Instrument types can represent stock, ETF, ETN, and future crypto without making crypto launch-active.
- Reference records have uniqueness constraints that prevent duplicate codes.

### Test Requirements
Test model creation, duplicate-code rejection, and future `CRY`/crypto representability without enabling crypto behavior.

### Dependencies
T2.

### Stop Condition
Stop when the reference schema is migrated and tested.
