## T16: Structured Logging and Metrics Plumbing

### Purpose
Add initial observability plumbing for structured logs and basic metrics counters/timers.

### Scope
Configure structured logging and metrics helpers for API start/stop, worker start/stop, job lifecycle events, admin-command events, and health-check failures.

### Allowed Changes
- Logging configuration
- Metrics helper modules
- Instrumentation calls in existing scaffold/job/admin/health boundaries
- Focused observability tests

### Forbidden Changes
- Do not build dashboards or production alert rules.
- Do not log provider-specific events beyond placeholder categories.
- Do not log credentials, tokens, provider keys, or raw secrets.

### Inputs
- T1 runtime roles
- T8 job state service
- T12 admin command schema
- T15 health checks

### Relevant Context
From Milestone 1, structured logs must include API/worker lifecycle, job creation/start/finish/failure/retry/cancel/stale-abandoned/lock events, admin commands, configuration changes, and health-check failures. Early metrics include API health, worker/scheduler heartbeat, database connectivity, queue depth, pending job age, job state counts, and API error rate.

### Implementation Budget
- Production files: expected maximum 2-5 closely related observability/config files plus tiny instrumentation edits.
- Behavior count: 1 behavior, emit structured observability events and basic metrics from existing boundaries.
- Model/API/workflow count: 0 models, 0 product APIs, 1 observability helper boundary.
- Test scope: log structure, required event names, secret redaction, and metric emission for existing boundaries.

### Expected Output
The platform emits structured logs and basic metrics hooks for the foundation built in Milestone 1.

### Acceptance Criteria
- Logs are structured and include stable event names.
- Job lifecycle and admin-command events can be logged through helpers.
- Health-check failures are logged without secrets.
- Basic metric helpers exist for health, queue age/depth, job state counts, and API error rate.

### Test Requirements
Test structured log fields, representative event emission, and secret/token redaction.

### Dependencies
T1, T8, T12, T15.

### Stop Condition
Stop when observability plumbing is in place and tested without dashboards or alerting.
