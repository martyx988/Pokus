## T7: Price Record Schema

### Purpose
Create the authoritative schema for selected daily price records.

### Scope
Add `PriceRecord` storage for adjusted historical closes and official exchange-local unadjusted current-day opens, including currency and selected-source evidence references.

### Allowed Changes
- Domain model files for price records
- Migrations for price tables and constraints
- Focused model tests

### Forbidden Changes
- Do not implement ingestion, candidate selection, or source-prioritization policy.
- Do not implement historical retention or reprocessing.
- Do not implement app price APIs.

### Inputs
- T4 instrument/listing schema
- T6 provider attempt schema
- Product requirements for daily adjusted close and current-day unadjusted open records

### Relevant Context
The system is daily-granularity only. Historical records use adjusted closes; current-day evaluation uses official exchange-local unadjusted opens. Every persisted price record must store currency. There must be a unique selected record per instrument/day/price type after later source prioritization.

### Implementation Budget
- Production files: expected maximum 1-2 closely related domain/migration files.
- Behavior count: 1 behavior, durable selected daily price storage.
- Model/API/workflow count: 1 model entity, 0 APIs, 0 workflows.
- Test scope: uniqueness, required currency, price type constraints, and relationships to listing/instrument/provider evidence.

### Expected Output
Price records can store daily selected values with currency, price type, exchange-local trading date, and source evidence.

### Acceptance Criteria
- Historical adjusted close and current-day unadjusted open price types are representable.
- Currency is required for each price record.
- Duplicate selected prices for the same instrument/listing, trading date, and price type are prevented.
- Provider evidence can be referenced without requiring candidate-value selection behavior.

### Test Requirements
Test valid price types, required currency, duplicate prevention, and provider-attempt reference behavior.

### Dependencies
T4, T6.

### Stop Condition
Stop when the price schema is migrated and tested without ingestion logic.
