## T45: Classify Instrument Opening Load Outcomes

### Purpose
Record terminal/non-terminal instrument outcome classes for opening loads, including missing, stale, halted, suspended, late-open, provider-failed, and success states.

### Scope
Implement per-instrument outcome classification and persistence only.

### Allowed Changes
- Instrument load outcome classification logic for opening-load workflow
- Outcome persistence fields and helpers for supported Milestone 4 cases
- Focused tests for outcome mapping rules

### Forbidden Changes

### Relevant Context
Publication readiness is exchange-scoped but depends on complete per-instrument outcomes. Correct terminal classification is required before coverage and readiness checks.

### Implementation Budget

- Behavior count: 1 behavior (classify and persist instrument opening-load outcomes).
- Model/API/workflow count: 1 outcome model family touchpoint, 0 APIs, 1 classification workflow.

### Expected Output

### Acceptance Criteria

- Terminal semantics align with FR-55c for publication-readiness usage.
- Outcome writes are stable under retries and reruns for the same load unit.
