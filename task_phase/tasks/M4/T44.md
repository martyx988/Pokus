## T44: Persist Selected Opening Price per Instrument-Day

### Purpose
Implement the per-instrument opening-load write path that turns provider candidates into one selected opening price record using existing source-prioritization behavior.

### Scope
Execute provider-fetch-to-selected-value-to-price-record flow for a single instrument-day opening load unit. No aggregate exchange-day state.

### Allowed Changes
- Opening-load worker behavior for per-instrument execution
- Existing candidate/selection and price persistence integration points
- Focused tests for selected opening-price storage behavior

### Forbidden Changes
- Do not compute exchange coverage or publication decisions.
- Do not classify final exchange/day readiness states.
- Do not expose app/operator APIs.

### Inputs
- T43 scheduled opening-load jobs
- Milestone 3 provider adapter contract, candidate evidence, and source prioritization outputs
- `design_phase/product_spec.md` FR-13, FR-17a

### Relevant Context
Milestone 4 trust flow needs a durable selected opening price per eligible instrument-day before outcomes and publication can be evaluated.

### Implementation Budget
- Production files: max 3-5 ingestion/pricing workflow files.
- Behavior count: 1 behavior (persist selected opening price for one instrument-day).
- Model/API/workflow count: 0-1 model touchpoints, 0 APIs, 1 per-instrument load behavior.
- Test scope: selected-value application to official opening-price record, idempotent rerun stability.

### Expected Output
Opening-load execution stores selected official exchange-local unadjusted opening prices for instrument-day units when candidate evidence exists.

### Acceptance Criteria
- Per-instrument opening-load path fetches normalized candidates and applies existing source prioritization.
- Selected opening value writes to price records with expected day/type identity.
- Re-running the same instrument-day load remains idempotent and does not create conflicting duplicate selected records.

### Test Requirements
- Tests for selected opening-price persistence from candidate inputs.
- Tests for idempotent rerun behavior for same exchange/instrument/day/load unit.

### Dependencies
- T43

### Stop Condition
Stop when per-instrument selected opening-price persistence is working and tested, without exchange-level aggregation.
