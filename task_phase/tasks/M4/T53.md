## T53: Build Operator Today-Opening Load Table Read Surface

### Purpose
Provide private operator read output for today-opening load status table with required columns, quality result, degraded flag, exception count, and timing.

### Scope
Implement operator query/read API or view-layer surface for today-opening table only.

### Allowed Changes
- Private operator read handler/service/serializer modules
- Query layer for required today-opening table fields
- Focused tests for table columns and visibility of blocked publication causes

### Forbidden Changes
- Do not implement yesterday-historical table in this task.
- Do not add manual retry/cancel/mark-failed command actions.
- Do not add broad dashboard features beyond today-opening table.

### Inputs
- T49 publication status records
- T50 read-model refresh outputs (or direct operator view model as currently implemented)
- `task_phase/roadmap_plan.md` M4 operator table acceptance criteria

### Relevant Context

### Implementation Budget
- Production files: max 2-4 operator read-surface files.
- Behavior count: 1 behavior (serve today-opening operator load table data).
- Model/API/workflow count: 0-1 read model/query family, 1 private operator API/view surface, 0 workflow changes.
- Test scope: required columns and blocked/failed/ready scenario rendering.

### Expected Output
Operator can retrieve today-opening load table rows with required status, counts, readiness, quality, degraded, exception, and timing fields.

### Acceptance Criteria
- Table output includes exchange, status, readiness, start/finish time, eligible count, success count, failure count, coverage, quality result, degraded flag, and exception count.
- Blocked correctness-validation situations are visible in operator-facing output.
- Output is private-surface only and respects existing auth boundaries.

### Test Requirements

- Tests for ready, blocked, failed, and market-closed row scenarios.
