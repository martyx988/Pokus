## T51: Expose Public Exchange Readiness Endpoints

### Purpose
Implement public API endpoints for exchange readiness reads using publication read models.

### Scope
Add readiness endpoint handlers/serializers only (`/exchanges/readiness` and `/exchanges/{exchange}/readiness/current` equivalents).

### Allowed Changes
- Public API readiness handlers, serializers, and routing
- Read-model query service used by readiness endpoints
- Focused endpoint tests for supported exchange readiness reads

### Forbidden Changes
- Do not expose current-day price payloads in this task.
- Do not change publication decision logic.
- Do not add private operator/admin endpoint logic.

### Inputs
- T50 refreshed readiness read model
- `design_phase/architecture.md` readiness endpoint definitions
- `design_phase/product_spec.md` readiness requirements

### Relevant Context
The mobile app first checks exchange readiness before requesting current-day prices, so this public surface must strictly reflect trusted publication state.

### Implementation Budget
- Production files: max 2-4 API/read handler files.
- Behavior count: 1 behavior (serve exchange readiness from read model).
- Model/API/workflow count: 0 models, 1 API surface group, 0 workflow changes.
- Test scope: readiness response correctness for ready/not-ready/market-closed states.

### Expected Output
Public readiness endpoints return exchange/day readiness states from publication read models.

### Acceptance Criteria
- Endpoint supports one or multiple launch exchanges as defined by API contract.
- Responses reflect read-model publication state rather than raw in-flight computation.
- Non-ready and blocked states are clearly represented.

### Test Requirements
- API tests for query variants and status values.
- Tests for market-closed and blocked-validation readiness outputs.

### Dependencies
- T50

### Stop Condition
Stop when readiness endpoints are implemented and tested, without current-day price endpoint work.
