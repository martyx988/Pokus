## T49: Decide Exchange-Day Publication Status

### Purpose
Apply publication state decision rules for opening-price exchange/day units: `ready`, `partial/problematic`, `failed`, `market closed`, and not-ready/in-progress.

### Scope
Implement publication decision state machine using aggregate status, terminal/coverage precheck, and correctness-validation result.

### Allowed Changes
- Publication decision module/state transition logic
- Publication record write/update behavior
- Focused tests for decision outcomes and fail-closed behavior

### Forbidden Changes
- Do not build API endpoints.
- Do not implement read-model refresh behavior.
- Do not add operator dashboard query logic.

### Inputs
- T46 aggregate exchange-day state
- T47 terminal+coverage precheck output
- T48 correctness-validation results
- `design_phase/product_spec.md` FR-26, FR-31a, FR-31b, AC-11

### Relevant Context
The app must only see ready exchanges; publication must fail closed. Exchange/day status classification must remain explicit for operator diagnosis.

### Implementation Budget
- Production files: max 2-4 publication decision files.
- Behavior count: 1 behavior (derive and persist final publication status per exchange/day opening load).
- Model/API/workflow count: 1 publication record family, 0 APIs, 1 decision workflow.
- Test scope: state mapping for ready/blocked/failed/market-closed/in-progress outcomes.

### Expected Output
Publication records are updated with correct exchange/day status and readiness flag based on milestone rules.

### Acceptance Criteria
- `ready` requires terminal outcomes, >99% coverage, and successful correctness validation.
- Market-closed path preserves market-closed status without price publication.
- Non-ready outcomes stay fail-closed for app-facing usage.

### Test Requirements
- Tests for each publication status outcome.
- Tests ensuring blocked validation prevents `ready`.

### Dependencies

### Stop Condition
Stop when publication decision logic is implemented and tested with no API/read-model changes.
